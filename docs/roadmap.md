# Roadmap

## Phase 1: MVP

- Map-first dashboard
- Mock incident data layer
- Source health visibility
- Laravel ingestion scaffolding
- Responsive command-center UI

## Phase 2: Operational Data

- Real ingestion from EONET, PAGASA, and PHIVOLCS
- Incident persistence and dedupe
- Advisory parsing and severity mapping
- Scheduler and queue orchestration
- Stale source handling

## Phase 3: Alerting

- Threshold-based alert engine
- SMS, email, and webhook notifications
- Region and hazard subscriptions
- Quiet hours and escalation policies

## Phase 4: Intelligence Layer

- Historical timelines
- Regional heatmaps
- Source confidence scoring
- Trend detection and anomaly rules
- Incident clustering and correlation
